## Routing pattern

Route files own auth guards via `beforeLoad`. The root route loads the session once; all other routes read `context.session?.user`.

```ts
// protected route
beforeLoad: ({ context }) => {
  if (!context.session?.user) throw redirect({ to: "/login" });
},
```

Never add `useEffect` redirect guards inside page components — the router handles it.

## Server state conventions

**Fetch hook** (`useFooFetch`) — wraps `useQuery` or `useQueries`, returns named aliases (`items`, `plan`, …) + `isLoading` + `error`.

**Actions hook** (`useFooActions`) — wraps one or more `useMutation` instances, invalidates affected query keys in `onSettled`. Returns named mutation objects so callers access `.mutate`, `.isError`, `.error`.

**Query keys** — all keys live in `src/lib/queryKeys.ts`. Use `planKeys.all` for broad invalidation, `planKeys.day(ts)` for a specific day.

## Plan draft lifecycle

`usePlanDraft(serverItems, mutate, dayTs, enabled)`:
- Syncs from `serverItems` when fingerprint changes (avoids clobbering in-progress edits).
- Debounces saves: 450ms after last change, fires `mutate`. Uses a `draftRef` snapshot to avoid stale closure.
- `flush()` cancels the pending timer and saves immediately (called on input blur via `onBlurCommit`).

`planBullets.ts` — all tree/flat operations are pure functions. Key types: `FlatBullet { key, text, completed, depth }`. `flatToPayload` trims empty rows, clamps depths, returns `PlanItemWrite[]`.

## Auth client

`src/lib/auth-client.ts` — single `authClient` instance. Intercepts every response to capture `set-auth-token` header into `localStorage`. Attaches stored token as `Authorization: Bearer` on every request. Used directly in `AuthContext` (for session reactivity) and in the root route `beforeLoad`.

## Mantine theme

Custom color `"ink"` (10-step purple). `primaryColor: "ink"`. `defaultRadius: "md"`. Font: `DM Sans`. Defined in `src/theme/index.ts`.
