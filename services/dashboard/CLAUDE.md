## Routing pattern

`_app.tsx` is a pathless layout route (TanStack Router `_` prefix). It guards all authenticated pages in one place and renders the shared header + `<Outlet />`.

```ts
// _app.tsx beforeLoad Рђћ single auth gate for all protected routes
beforeLoad: ({ context }) => {
  if (!context.session?.user) throw redirect({ to: "/login" });
},
```

Never add `useEffect` redirect guards inside page components Рђћ the router handles it.

## Header layout

Two-row sticky header in `_app.tsx`:
- **Row 1** (always, h=64): logo ┬и feature tabs (Day Planner / Knowledge) ┬и user email + logout
- **Row 2** (only on `/knowledge*`, h=44): Knowledge sub-nav links (Add Entry / Browse)

`isKnowledge = location.pathname.startsWith("/knowledge")` drives both the active tab highlight and the second row visibility. Total header height: 64px (planner) or 108px (knowledge).

## Server state conventions

**Fetch hook** (`useFooFetch`) Рђћ wraps `useQuery` or `useQueries`, returns named aliases (`items`, `plan`, Рђд) + `isLoading` + `error`.

**Actions hook** (`useFooActions`) Рђћ wraps one or more `useMutation` instances, invalidates affected query keys in `onSettled`. Returns named mutation objects so callers access `.mutate`, `.isError`, `.error`.

**Query keys** Рђћ all keys live in `src/lib/queryKeys.ts`. Use `planKeys.all` for broad invalidation, `planKeys.day(ts)` for a specific day. `knowledgeKeys.all` for all knowledge entries.

## Plan draft lifecycle

`usePlanDraft(serverItems, mutate, dayTs, enabled)`:
- Syncs from `serverItems` when fingerprint changes (avoids clobbering in-progress edits).
- Debounces saves: 450ms after last change, fires `mutate`. Uses a `draftRef` snapshot to avoid stale closure.
- `flush()` cancels the pending timer and saves immediately (called on input blur via `onBlurCommit`).

`planBullets.ts` Рђћ all tree/flat operations are pure functions. Key types: `FlatBullet { key, text, completed, depth }`. `flatToPayload` trims empty rows, clamps depths, returns `PlanItemWrite[]`.

## Auth client

`src/lib/auth-client.ts` Рђћ single `authClient` instance. Intercepts every response to capture `set-auth-token` header into `localStorage`. Attaches stored token as `Authorization: Bearer` on every request. Used directly in `AuthContext` (for session reactivity) and in the root route `beforeLoad`.
